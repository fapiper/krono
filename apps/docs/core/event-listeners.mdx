---
title: 'Orderbook Event Listeners'
description: 'Overview of all event subscription methods in the Orderbook class.'
---

The `Orderbook` class provides high-level helper methods to subscribe to data updates, status changes, and errors. These listeners are the primary way to integrate the orderbook into your UI or data processing logic.

## Core Listeners

- [onData](/core/api/orderbook/onData): Triggers on processed (throttled/debounced) updates.
- [onStatusUpdate](/core/api/orderbook/onStatusUpdate): Triggers when connection state changes.
- [onError](/core/api/orderbook/onError): Triggers when an error occurs.

## Internal Listeners

- [onRawDataUpdate](/core/api/orderbook/onRawDataUpdate): Direct socket updates (high frequency).
- [onHistoryUpdate](/core/api/orderbook/onHistoryUpdate): Triggers when a snapshot is archived.
- [onConfigUpdate](/core/api/orderbook/onConfigUpdate): Triggers when settings change.
