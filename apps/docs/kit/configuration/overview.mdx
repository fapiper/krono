The `Orderbook.Root` component serves as the context provider for all orderbook-related components in `@krono/kit`. It wraps the core `@krono/core` functionality and provides React hooks and components with access to orderbook data, asset pairs, and playback features.

## Basic Setup

The minimal configuration requires only a trading pair symbol:

<CodeGroup>
    ```tsx Basic
    import { Orderbook } from '@krono/kit'

    export const App = () => {
    return (
    <Orderbook.Root config={{ symbol: 'BTC/USD' }}>
    {/* Your orderbook components */}
</Orderbook.Root>
)
}
```
```tsx With Components
import { Orderbook, OrderbookTable } from '@krono/kit'

export const App = () => {
return (
<Orderbook.Root config={{ symbol: 'BTC/USD' }}>
    <Orderbook.Panel
        renderTable={(props) => <OrderbookTable.Root {...props} />}
    />
</Orderbook.Root>
)
}
```
</CodeGroup>

## Configuration Structure

The `config` prop accepts an object with orderbook settings and optional asset pairs configuration:

<CodeGroup>
    ```tsx Basic
    <Orderbook.Root
        config={{
            symbol: 'BTC/USD',
            depth: 100,
            limit: 50
        }}
    >
        {/* Components */}
    </Orderbook.Root>
    ```
    ```tsx With Asset Pairs
    <Orderbook.Root
        config={{
            symbol: 'BTC/USD',
            depth: 100,
            limit: 50,
            assetPairs: {
                topN: 100,
                autoFetch: true
            }
        }}
    >
        {/* Components */}
    </Orderbook.Root>
    ```
    ```tsx Complete
    <Orderbook.Root
        config={{
            symbol: 'BTC/USD',
            depth: 100,
            limit: 50,
            throttleMs: 100,
            historyEnabled: true,
            maxHistoryLength: 50,
            reconnect: {
                enabled: true,
                maxAttempts: 5,
                delayMs: 1000
            },
            assetPairs: {
                topN: 100,
                autoFetch: true
            }
        }}
    >
        {/* Components */}
    </Orderbook.Root>
    ```
</CodeGroup>

## Context Providers

The `Orderbook.Root` component automatically provides three context providers:

<Steps>
    <Step title="AssetPairsProvider">
        Manages trading pair metadata including tick sizes, lot sizes, and pair information.

        Accessible via the `useAssetPairs()` hook from `@krono/hooks`.
    </Step>

    <Step title="OrderbookProvider">
        Handles WebSocket connection, orderbook data streaming, and reconnection logic.

        Accessible via the `useOrderbook()` hook from `@krono/hooks`.
    </Step>

    <Step title="OrderbookPlaybackProvider">
        Enables historical snapshot tracking and playback features.

        Accessible via the `useOrderbookPlayback()` hook from `@krono/hooks`.
    </Step>
</Steps>

All child components can access these contexts through the provided hooks.

## Props

<ParamField path="config" type="OrderbookRootConfig" required>
    Configuration object for orderbook and asset pairs.

    <Expandable title="config properties">
        <ParamField path="symbol" type="string" required>
            The trading pair symbol (e.g., `BTC/USD`, `ETH/EUR`).
        </ParamField>

        <ParamField path="assetPairs" type="AssetPairsConfig">
            Optional asset pairs configuration. See [Asset Pairs Configuration](/kit/configuration/asset-pairs).
        </ParamField>

        <ParamField path="...orderbookConfig" type="OrderbookConfig">
            All other orderbook configuration options. See [Orderbook Configuration](/kit/configuration/orderbook).
        </ParamField>
    </Expandable>
</ParamField>

<ParamField path="children" type="ReactNode" required>
    Child components that will have access to the orderbook context.
</ParamField>

## Notes

<Note>
    The `symbol` property is the only required configuration field. All other settings have sensible defaults.
</Note>

<Tip>
    Enable `debug` mode during development to see detailed logging of connection events, updates, and errors.
</Tip>

<Warning>
    Changes to the `config` prop will reinitialize the orderbook connection. Use stable configuration objects to avoid unnecessary reconnections.
</Warning>
